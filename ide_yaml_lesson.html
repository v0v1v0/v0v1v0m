4  Guide to Writing YAML Lessons for the Numerious IDE

4.1 Structure of a Lesson

Each lesson consists of multiple blocks, which can include the following sections:

4.1.1 Sections

  • html: Contains HTML content for modals or additional context.
  • question: Instructions displayed in the R Console, written in the voice of R-bot.
  • code: The correct R code to solve the block.
  • hints: Progressive hints to assist users, also written in R-bot’s voice.
  • feedback: Encouraging feedback displayed upon block completion.
  • answers: Variables or expected outputs to validate user input.

4.2 String Formatting with {{#tag}}...{{/tag}}

Use {{#tag}}...{{/tag}} to format console output. This simplifies readability for non-technical writers.

4.2.1 Supported Tags

Tag Purpose Example
green Highlight important instructions or terms in green. {{#green}}Remember to save your work!{{/green}}
var Bold variables to draw attention. Assign the value to {{#var}}x{{/var}}.
func Italicize and cyan color for functions. Call {{#func}}print(){{/func}} to see the result.
code Monospaced text for R code snippets. Type {{#code}}x <- 42{{/code}} in the console.
file Bold and italicized for filenames or paths. Save to {{#file}}./out/results.csv{{/file}}.

4.3 Writing YAML Sections

4.3.1 HTML Section

Use HTML for visuals and context. Avoid using {{#tag}} syntax here; use HTML styling instead.

html: |
  <div>
    <h2>Welcome to Numerious Inc.</h2>
    <p>Here’s an overview of your first task.</p>
  </div>

4.3.2 Question Section

Provide instructions using {{#tag}} formatting for clarity.

question:
  - "To confirm your role, assign your job title to {{#var}}role{{/var}}."
  - "Example: {{#code}}role <- 'Data Analyst'{{/code}}"

4.3.3 Code Section

Write valid R code that solves the block.

code:
  - "role <- 'Data Analyst'"

4.3.4 Hints Section

Write progressive hints to guide the user.

hints:
  - "Assign {{#green}}'Data Analyst'{{/green}} to {{#var}}role{{/var}}."
  - "The correct syntax is {{#code}}role <- 'Data Analyst'{{/code}}."

4.3.5 Feedback Section

Encourage users with feedback upon completion.

feedback: "Great job! You’ve successfully assigned {{#var}}role{{/var}}. Let’s move on."

4.3.6 Answers Section

List variables or outputs the system should validate.

answers:
  - "role"

4.4 Example Block

Here’s a complete example of a YAML block:

block1:
  html: |
    <div>
      <h2>Welcome to Numerious Inc.</h2>
      <p>Let's get started with your first task!</p>
    </div>
  question:
    - "Assign your job title to {{#var}}role{{/var}}."
    - "Example: {{#code}}role <- 'Data Analyst'{{/code}}"
  code:
    - "role <- 'Data Analyst'"
  hints:
    - "Think of your job title. Assign it to {{#var}}role{{/var}}."
    - "Use this syntax: {{#code}}role <- 'Data Analyst'{{/code}}."
  feedback: "Excellent work! You’ve successfully assigned {{#var}}role{{/var}}."
  answers:
    - "role"

4.5 Guidelines for Non-Technical Writers

  • Focus on Clarity: Write instructions and hints as if explaining to a beginner.
  • Use Tags Sparingly: Use {{#tag}}...{{/tag}} only when needed.
  • Test Your Code: Ensure the code section works in R before submitting.
  • Encourage Progress: Keep feedback positive and motivating.
  • Be Consistent: Follow the same structure and formatting across lessons.

This guide ensures a structured, user-friendly approach to writing YAML courses.
